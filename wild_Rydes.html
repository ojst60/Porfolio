This was an important and interesting project because it helped me to understand how AWS services can be combined to create a multi-tier architecture.

The beginning of this project started by creating an AWS cloud9 environment which is AWS IDE. Using AWS cloud9, I was able to create a repository containing the source code for the static part of the website with AWS code commit. AWS code commits is amazon's version of github. I deployed the website uing AWS amplify by coneecing the source code from aws code commit git repository. AWS amplify also supports continuous integration and continuous delivery workflow which are important in making changes .

In order for users to register and login to the worldwide service, I created a user pool using Amazon Cognito. Amazon Cognito handles the user verification, user authentication and account recovery of the webapp users

The final part of the project was to create the backend part of the webapp by using DynamoDB and AWS lambda. Dynamodb is AWS schema-less database that requires only a primary key. I created a dynamoDB table called rides and gave it a primary key called ride ID. I also created a lambda function that is invoked when a user requests a ride. The function reacts to the user's request by selecting a unicorn from the fleet and recording the request on the dyanmoDB table. For the backend process to be successful, I created a restful API using amazon API gateway and attached a role to the lambda function with an AWSlambda execution policy and dynamoDB: put inline policy.
